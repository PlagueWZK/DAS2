# 王忠睿答辩文档（数据增强算法）

## 负责范围
- 数据增强算法实现（`data_augmentation.py`）
- OpenCV 与 Pillow 的图像处理流程
- 增强方法组合与参数控制

## 关键实现要点
- 旋转、翻转、裁剪、缩放保证输出维度与原图一致
- 颜色抖动使用 Pillow 调整亮度、对比度、饱和度
- 噪声与模糊基于 OpenCV 实现，适配多种输入类型
- `apply_augmentation()` 支持按列表灵活组合增强方式

## 可能问题与回答

### Q1：实现了哪些增强方法？
A：旋转、翻转、裁剪、缩放、颜色抖动（亮度/对比度/饱和度）、亮度调节、对比度调节、噪声、模糊。

### Q2：为什么需要保持输出尺寸与原图一致？
A：前端对比预览和批量下载要求统一尺寸，且模型训练通常需要固定输入大小，保持尺寸能减少后续处理成本。

### Q3：为何颜色抖动使用 Pillow 而非 OpenCV？
A：Pillow 在颜色增强方面 API 更直接，且可读性高；最终再转换回 OpenCV 格式保证后续一致处理。

### Q4：随机裁剪/缩放如何保证图像内容有效？
A：随机比例控制在合理区间（如 0.7~0.9），缩放后按中心裁剪或居中填充，避免全黑或过度裁剪。

### Q5：噪声与模糊的实现细节？
A：噪声支持高斯与椒盐，模糊支持高斯与中值滤波，接口预留参数以便前端扩展。

### Q6：增强链路是否可扩展？
A：`apply_augmentation()` 根据列表顺序执行，可快速添加新方法并通过前端选项组合。

### Q7：OpenCV 的 BGR 与 Pillow 的 RGB 如何处理？
A：在 OpenCV 与 Pillow 之间转换时使用 `cv2.cvtColor`，保证颜色通道正确。

### Q8：增强参数范围如何确定？
A：范围基于常见数据增强经验值（如旋转角度、裁剪比例），以保证视觉合理性；如需更强增强可在参数中调整。

### Q9：增强顺序会影响结果吗？
A：会影响。系统按列表顺序执行，先裁剪再缩放与先缩放再裁剪会产生不同效果，便于按任务需求调整。

### Q10：如何保证增强后图像维度一致？
A：裁剪后会 resize 回原尺寸，缩放后按中心裁剪或居中填充，确保输出与输入一致。

### Q11：随机性如何复现？
A：当前默认随机，若需要复现可在运行前设置随机种子（`random`、`numpy`），后续可扩展为前端参数。

### Q12：对灰度图或带透明通道的支持如何？
A：当前以三通道图像为主，灰度/透明图可在读取时做通道转换或统一为三通道，后续可扩展兼容。
